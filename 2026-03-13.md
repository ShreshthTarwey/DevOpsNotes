## 1. `history` (PowerShell Command)

**Command:** `history`

**Purpose:** Displays the command history in PowerShell.

**Analysis:**
- Shows previously executed commands with line numbers
- Useful for recalling and reusing commands

**Key Insights:**
- PowerShell's built-in history functionality
- Different from `docker history` which shows image layers

## 2. `docker history` (Error Case)

**Command:** `docker history`

**Analysis:**

**Key Insights:**
- Demonstrates that `docker history` needs an image parameter

## 3. `docker run lpu`

**Command:** `docker run lpu`

**Purpose:** Attempts to run a container using "lpu" as the image name.

**Analysis:**
- "lpu" was previously used as a container name, not an image name

## 4. `docker run httpd`

**Command:** `docker run httpd`

**Purpose:** Runs a container from the httpd image in foreground mode.

**Analysis:**
- Starts Apache HTTP server in the foreground
- Container runs until manually stopped (Ctrl+C)
- Exposes port 80 by default

**Key Insights:**
- Foreground containers block the terminal
- httpd image runs a web server by default

## 5. `docker run -d httpd`

**Command:** `docker run -d httpd`

**Purpose:** Runs httpd container in detached (background) mode.

**Analysis:**
- `-d` flag runs container in background
- Returns container ID immediately
- Container continues running after command completes

**Key Insights:**
- Detached mode allows multiple containers to run simultaneously
- Useful for long-running services like web servers

## Summary of Key Docker Concepts Demonstrated

1. **Container Lifecycle:** run → stop → start → rm
2. **Detached vs Foreground:** `-d` flag for background execution
3. **Container Management:** `docker ps`, `docker ps -a`, `docker rm`
4. **Force Operations:** `docker rm -f` for forceful removal
5. **Interactive Containers:** `-it` flags for shell access
6. **Bulk Operations:** Removing multiple containers at once
7. **Shell Integration:** WSL and different shell environments

## Practical Takeaways

- Detached mode (`-d`) allows containers to run in background
- Container names are auto-generated and unique
- `docker ps -a` shows all containers, `docker ps` shows only running
- Interactive containers (`-it`) provide shell access
- Force removal (`-f`) bypasses safety checks
- Different shells have different command syntaxes
- Command history is valuable for learning and troubleshooting
